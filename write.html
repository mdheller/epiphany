Write on Epiphany

Epiphany has a similar concept like Jupyter notebook where an aritcle is formed by a few blocks or cells. A cell can either be a piece of text content or a runnable program. In the future, Epiphany may support more block types than just text and program.

To create a new post, once logged in, navigate to the profile page and click on the New Post button:

A blank post should be generated with an empty Meta block you need to fill in. The first block of any article is always a Meta block. This is a special block that you cannot create, delete or move. It is generated automatically when creating a new post. A post must have one Meta block as the first block.

You need to first finish this meta block by providing a title and optionally some tags. Once ready, click the Play button at the bottom right, this action will generate a new post for you.

Now, a new post is formally created with the Meta block at the top. You may notice that if you move your mouse to a block, it will be highlighed with blue dash lines on both sides, meanwhile, a few buttons will appear. These buttons are the edit button. You can use them to toggle between the editing mode and the viewing mode, remove a block, insert a block or reorder blocks.

Now, since we have no blocks in this empty post, the first step is pressing the Plus button to add a new block. A newly added block has no type.

You need to provided the desired block type through the dropdown and then confirm by clicking Add.

We will cover how to write a program with Epiphany in the next session. For now, let's choose Markdown.

Once a Markdown block is created, a placeholder line is created. You can now write your block content. To do this, just click the Edit button.

This will switch the current block into the edit mode. In the edit mode, you will have a markdown editor as well as the rendered markdown result.

If you like a more focused writing experience, you can also switch to the fullscreen editing mode. In this mode, only the editor and the rendered result are displayed side by side. Once done editing, click on the Play button, the block will switch to the viewing mode again and the article is saved to the server at the same time.

Click the Edit button of the Meta block again. Here you will see 2 buttons. The Publish/Unpublish button toggles the visibility of this article, whereas the Commit button save the current version to the published version.

Any post has two versions, a Draft version and a Published version. Any modification an author makes will only apply to the draft version. Only when committed, the modification made to the draft will be copied to the published version.

Internally, Epiphany adopts git for storage and version control. And editing to the draft will go into the staging area of the post's git repository. Only when the author triggers commit, the draft will be committed to the git repository.

An author always sees the latest draft of her post, whereas other users will see the latest published version. If an author wants to hide an article, she just needs to toggle the state to unpublished.
