Write on Epiphany

On Epiphany, an article is formed by a few blocks. A block can either be a piece of text or a runnable program. We currently support using Markdown for text content, and two languages, Javascript and Python for programming. In the future, Epiphany will support more languages and block types.

To create a new article, once logged in, navigate to the profile page. The profile page is where you can manage all of your articles. Click on the "+ New Post" button:

A blank post should be generated with an empty title. Like previously said, an article is formed by a series of blocks. Hence the title also belongs to a block called a Meta block. This block contains all meta data related to a post, including a title, a few tags, an abstract and a thumbnail image. An article has one and only one Meta block as its first block. This is a special block that you cannot delete or move. It is generated automatically when creating a new post.

Please provide a title and optionally some tags. Once ready, click the Play button at the bottom right, this action will generate a new post for you.

Now, a new post is formally created with only the Meta block at the top. Every block can appear in one of the two modes, the viewing mode and the editing mode. In the viewing mode, the reader can't change the block's content. All UI related to editing will be hidden. To make modification to a block, we need to switch to the editing mode first. You may have noticed that if you hover mouse over a block, the block will be highlighted with blue dash lines on both sides. Meanwhile, a toolbar will appear on the bottom right corner. This is the toolbar you can use to toggle between the editing mode and the viewing mode, remove the block, insert a new block or reorder blocks.

Since we have no blocks other than the Meta block, the first step is adding a new block with the "+" button.

The new block has no type. Select a type through the dropdown and then confirm by clicking Add.

We will cover how to write a program in the next session. For now, let's choose Markdown.

To modify block content, click the Edit button to switch the current block to the editing mode.

In the edit mode, you will see the editor. The rendered markdown will be displayed underneath.

If a more focused writing experience is preferred, you can switch to the full-screen mode. The editor and the rendered result are displayed side by side. Once done editing, click on the Play button to apply the change. The block will switch to the viewing mode again and the article is saved to the server as a draft. All changes you have made to a block will first be saved to the draft. To finalize your change, you need to commit it. This work-flow is similar to that of the Git version control. The draft is only visible to you and stays in a staging area, whereas a reader of your article will only see the last committed change.

To commit the current change, click the Edit button of the Meta block again. Here you will see two buttons. The Commit button commit the draft. The Publish/Unpublish button toggles the visibility of the article. Only when an article is in the published state, it becomes accessible to others and is publicly listed in your profile page.

If you want to pull an article off from public accessing, you can change the article's state back to unpublished. It will become private again.
